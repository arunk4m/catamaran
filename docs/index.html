Built for two-cluster problems

Everything a Kubernetes IDE does — with depth perception.

Dev vs prod, blue vs green, old vs new: operating Kubernetes is a comparison workload. Catamaran makes the comparison a first-class layout instead of a pile of windows.

⛵

Split View deck

Two independent panes — each with its own context, namespace, tabs, and log dock. Linked cruising mirrors navigation across both while each keeps its own cluster.

📡

Live everything

40+ resource kinds streamed by server-side watches — no polling. Logs follow with reconnect, timestamps, since/tail windows, and previous-instance reads.

🤖

MCP-native

Every backend capability is also an MCP tool — 66 and counting — so agents operate clusters through the same audited code paths as the UI, with destructive ops gated.

🔭

Spyglass: Kiali & Grafana

One click auto-discovers the tool in your cluster, holds a managed port-forward open, and launches it in a dedicated window — mesh topology and dashboards without leaving Catamaran.

🪶

Rust-light

Pure-Rust core on kube-rs and tokio inside a Tauri v2 shell. A ~12 MB DMG that talks straight to your API servers — no Node main process, no bundled Chromium.

⌨️

Keyboard-first

A ⌘K palette with fuzzy + frecency ranking that jumps to any view or resource and runs workspace actions — split, link, swap, theme — without leaving home row.

🛡️

Safe by default

Confirmation-gated destructive actions, YAML editing with schema validation, and exec-auth hardening so stale terminal credentials can never hijack your identity.

Agents welcome aboard

The GUI and your AI agent share one capability registry.

Run the same binary as an MCP server over stdio or loopback HTTP. Tools carry read-only/destructive annotations, and nothing destructive runs without an explicit confirmation argument.

# stdio transport (local agents / IDE clients)
catamaran --mcp-stdio

# loopback HTTP (default 127.0.0.1:8765)
catamaran --mcp-http

Set sail

Install in under a minute.

Download the DMG

Grab Catamaran_<version>_aarch64.dmg (Apple Silicon) or _x64.dmg (Intel) from the latest release.

Drag to Applications

Community builds aren't Developer-ID signed yet — first launch: right-click → Open, or run xattr -cr /Applications/Catamaran.app once.

Open a context, then split the deck

Catamaran reads your kubeconfig (including exec auth for EKS/GKE/AKS). Hit ⌘\, put a second context beside the first, and compare away.
